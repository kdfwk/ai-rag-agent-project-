# 长期记忆功能实现文档

## 📋 功能概述

本次更新为智扫通机器人智能客服添加了**真正的长期记忆能力**，使其能够：
- ✅ **记住用户姓名**（如"我叫小帅"）
- ✅ **记住用户偏好**（如"我喜欢木地板"、"我家有宠物"）
- ✅ **跨会话记忆**（重启服务器后依然记得）
- ✅ **个性化回答**（根据历史对话提供定制化服务）


**作用**：让 RAG 系统在检索产品知识的同时，也能考虑用户的个人情况，提供更个性化的答案。

## 🔍 技术细节

### 1. 向量相似度搜索
使用 ChromaDB 的 `similarity_search` 方法，通过嵌入模型（Embedding Model）计算语义相似度：

```python
docs = self.vector_memory.similarity_search(query, k=k)
```

**优势**：即使用户用不同的措辞，也能找到相关记忆。
- 用户说："我家主子掉毛厉害"
- 系统能匹配到："我家有只猫"

### 2. 记忆去重与优先级
- **k 值控制**：每次最多检索 3 条相关记忆，避免上下文过长
- **时间衰减**：最新的对话会被优先检索（ChromaDB 默认按插入顺序）

### 3. 异常处理
所有记忆检索操作都包裹在 `try-except` 中，确保即使向量库出现问题，也不会影响正常对话：

```python
try:
    long_term_memories = memory_service.search_vector_memory(query, k=3)
except Exception as e:
    long_term_memories = []  # 降级为空列表
```

## ⚙️ 配置说明

### 向量数据库位置
```

├── chroma.sqlite3          # SQLite 数据库文件（记忆存储在这里）
└── <collection_id>/        # 向量索引文件
```

### 记忆集合名称
在 `config/chroma.yml` 中配置：
```yaml
memory_collection_name: "agent_memory"
```

### 检索数量
在代码中硬编码（可根据需要调整）：
- `chat_node`: k=3（闲聊时检索 3 条记忆）
- `rag_node`: k=2（专业问答时检索 2 条记忆）

## 🛠️ 维护与优化

### 清空记忆
如果需要重置所有记忆：
```bash
# 删除向量数据库
rm -rf chroma_db/

# 重新构建知识库
python -m rag.vector_store
```

### 查看记忆内容
可以使用 ChromaDB 的官方工具或编写脚本查询：
```python
from agent.memory import memory_service
memories = memory_service.search_vector_memory("", k=100)
for m in memories:
    print(m.page_content)
```

### 性能优化建议
1. **定期清理**：如果对话量很大，可以定期删除过时的记忆
2. **增加 k 值**：如果发现记忆不够用，可以增加检索数量（但会增加 token 消耗）
3. **优化提取规则**：在 `extract_facts` 中添加更多关键词触发器

1. **真正的持久化**：记忆存储在硬盘上的 SQLite 数据库，重启不丢失
