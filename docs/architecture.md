# 架构设计说明

## 文档处理架构

### 概述
系统采用分层架构处理不同类型的文档，确保职责分离和模块化。

### 模块职责划分

#### 1. `rag/multimodal_ocr.py` - OCR服务层
- **职责**：专门处理图像识别和OCR功能
- **能力**：
  - 单张图片OCR识别 (`recognize_image`)
  - PDF页面OCR识别 (`recognize_pdf_pages`)
  - 统一使用视觉大模型进行文字识别
- **输入**：图片路径、PDF路径
- **输出**：纯文本内容

#### 2. `utils/document_loader.py` - 文档解析层
- **职责**：解析各种文档格式，整合文本和OCR结果
- **能力**：
  - PDF文档解析：提取原生文本 + 调用OCR服务识别扫描内容
  - Word文档解析：提取文字内容 + 提取并OCR内嵌图片
  - 图片文档解析：直接调用OCR服务
  - TXT文档解析：纯文本读取
- **输入**：文档路径
- **输出**：LangChain Document对象列表

### 架构优势

1. **单一职责原则**：OCR服务专注于图像识别，文档解析专注于格式处理
2. **复用性**：不同格式的文档都可以复用统一的OCR服务
3. **维护性**：OCR相关改动只需在一处修改
4. **可测试性**：各层功能独立，便于单元测试

### 处理流程

#### PDF文档处理
1. 使用PyPDFLoader提取原生文本层
2. 调用OCR服务对扫描页面进行识别
3. 合并两种结果并返回

#### Word文档处理  
1. 使用python-docx提取文字内容
2. 提取文档内嵌图片
3. 调用OCR服务对图片进行识别
4. 合并文字和OCR结果

#### 图片文档处理
1. 直接调用OCR服务进行识别
2. 返回识别结果

### 扩展性
- 新增文档格式：在document_loader中添加解析函数
- 新增OCR能力：在multimodal_ocr中扩展方法
- 更换OCR引擎：只需修改multimodal_ocr实现
